# 暑期排班系統｜Shift 1 模組

A Plus 教學部暑期班（8/3 ～ 8/28，週一至週五）排班設定工具，Phase 2 版本。

🔗 **線上網址**：`https://你的帳號.github.io/你的repo名稱/`（部署後請自行補上）

---

## 功能概覽

- **教師管理**：
  - 新增／編輯教師資料，可設定週時數目標、不可排班（封鎖）時段
  - **固定教室（homeroom）**：可指定教師在正課時段固定使用的教室（已內建 12 位教師的預設對應），未指定則視為彈性教師（例如代課）
  - **星期分組工作班型**：可依「星期一、三」「星期二、四」「星期五」三組分別設定不同班型，同一位老師不同天可以有不同上班時間
- **教室管理**：
  - 設定可用教室與容納人數
  - **使用教師欄位**：顯示該教室固定使用的教師姓名；廣場類教室（1廣／2廣／3廣）標示為「托育輪值共用」
- **時段設定**：
  - 拖曳排序調整時段順序
  - 依班型（Shift）套用預設時段組合
  - 每個時段可設定類型（正課／其他）、備注
  - 自動計算每個時段時數，時間錯誤會即時警示
- **營隊分組管理**：下午時段可依營隊拆分多個分組，方便安排人力
- **排班編輯**：
  - 一週格狀班表：教室 × 時段，下拉選單直接指派教師
  - 週次／星期切換（週一～週五，共 5 天）
  - **教師衝突偵測**：同一教師同時段被排入多間教室時，儲存格標紅並跳出警示
  - **封鎖時段偵測**：指派到教師本身設定的不可排班時段時，儲存格標黃提示
  - **教室規則自動鎖定**：07:00–09:00、16:30–18:30（托育時段）只能選廣場；09:00–16:30（正課核心時段）只能選年級教室，不適用的組合會直接鎖住顯示「不適用」
  - **班型＋固定教室連動篩選**：每格下拉選單只列出「當天班型時段涵蓋」且「教室規則相符」的教師（有固定教室的老師只會出現在自己教室）
  - **一鍵批次帶入**：選擇教師後，系統依「當天所屬星期分組的班型」自動判斷時段，並自動填入正確教室——固定教室老師自動進自己教室，托育時段自動導向對應樓層廣場，彈性教師（無固定教室）才需另選備援教室
- **教師檢視**：
  - 選擇教師＋週次，唯讀顯示週一～週五完整班表
  - 自動加總本週已排時數，並依教師的週時數上下限標示「未達下限／正常範圍／超過上限」
- **資料匯出／匯入**：可將目前設定（含教師、教室、時段、營隊、排班資料）匯出成 JSON 檔備份，或匯入還原

## 資料儲存方式（重要）

目前版本是**純前端靜態頁面**，所有資料儲存在使用者瀏覽器的 `localStorage` 中：

- ⚠️ **資料不會同步**：不同人、不同裝置、不同瀏覽器看到的資料是各自獨立的，並非共用同一份班表
- ⚠️ **清除瀏覽器資料會遺失設定**：建議定期使用「匯出」功能備份成 JSON 檔
- 若多人需要共同編輯同一份班表，需要接後端資料庫（例如 Firebase、Google Sheets API），屬於後續規劃項目
- 系統內建自動資料遷移：舊版儲存的教師資料（單一班型欄位）會自動轉換為星期分組格式，不需要手動重新設定

## 部署方式（GitHub Pages）

1. 檔案需命名為 `index.html`（全小寫）
2. Repo 需為 Public
3. Settings → Pages → Source 選 `Deploy from a branch` → Branch 選 `main` → 資料夾選 `/ (root)`
4. 儲存後約 1–2 分鐘內生效
